Introduction to WebGL

This page provides an introductory lesson to WebGL with a focus on providing you with the basic tools to building a fully functioning Pong game.

Skip to the Demos!

Learning Objectives

  • Learn what you will need to know to complete this lesson.
  • Ensure your browser is WebGL capable.
  • Learn about the files used by the WebGL demos. You will need this knowledge for the proceeding sections.
  • Understand the benefits/disadvantages of WebGL.
  • Learn about the history of WebGL.
  • Gain an understanding of the WebGL Pipeline.
  • Be able to explain the purpose and utilization of Vertex and Fragment shaders.
  • Be able to describe and apply the basic steps to initialize a WebGL application.
  • Be able to describe and apply the basic steps required to draw a basic primitive in WebGL.
  • Describe the Cartesian coordinate space conventionally used by 2D Canvas.
  • Describe the Cartesian coordinate space conventionally used by the WebGL Canvas.
  • Understand the difference between Model, View/Camera and Projection Spaces.
  • Understand how to convert between Model, View/Camera and Projection Spaces.
  • Describe the two popular types of 3D projection used in WebGL.
  • Understand the basics of lighting in your WebGL application.
  • Apply your understanding of this lesson to write your own WebGL applications!

If you are still good to go - the next step to mastering WebGL (and all the wonders that go along with it!) is to understand the technology and composition of the demos that you will be working with moving forward. The following sections will provide you with an overview of the demo files composition (i.e. where you can find the demo code, areas where you can add your own demo code, etc.) and 3rd party libraries used in the demos. Let's start with the 3rd party libraries and finish up with the demo code.
  • 3rd Party Libraries Used in the Demos
    • jQuery: Used as a cross-browser solution to gain access to the DOM, used to build the canvas widget that displays the demos. It is also used to make the magic of this site work.
    • AngularJS: This a Model/View/Controller JavaScript framework used to organize your code. All demos are placed within the AngularJS controller scope. See WebGLMainController.js to view the main controller that runs the WebGL demos in this site. Note that you do not need to understand how AngularJS works to be able to play around with the demo code (or write your own). See the following "JavaScript Demo Files Composition" section for more details.
    • glMatrix: We use this library to create the necessary mathematical data structures (matrices and vectors) and data operations (matrix multiply, transpose, invert, etc.). This lightweight library can also generate the appropriate Model, View/Camera and Projection matrices that the WebGL subsystem requires (so we don't have too) to do it's magic!
  • JavaScript Demo Files Composition
    • scripts/angular-app/app.js: This is the entry point to the angularJS app used. This file defines the primary module (and dependencies required) that will be used for the site. There will be no need to modify the code in this file.
    • scripts/angular-app/lightingExample.js: This file contains a WebGL lighting example that uses the Lambert Shading technique to make our 3D objects look good. This is an advanced example and should only be looked at after you grasp the fundamentals.
    • scripts/angular-app/pong.js: This file contains the Pong game. The code is not 100% efficient and there is room for improvement (an exercise for you perhaps??) but it is fairly structured and commented and will serve as a good learning point for you. This is an advanced example and should only be looked at after you grasp the fundamentals.
    • scripts/angular-app/controllers/WebGLMainController.js: As was mentioned earlier, this file contains the primary AngularJS controller used for this site. It features the first 6 demos and is the area that you will be most interested in. This is the file you will need to modify to add in your own examples. Look for the phrase "STUDENT EXERCISES!!" in this file to see how this can be achieved.
    • scripts/angular-app/services/WebGLUtilityModule.js: This file provides the backbone for displaying the demos. The code is contained in an AngularJS Service (you can think of it as being a singleton object available only to AngularJS apps). Basically the code in this service creates a modal window with two HTML5 canvases. A WebGL canvas that runs our demo code, and a 2D canvas which is used as a Heads Up Display (HUD) to provide an overlay for our score and title menu in the Pong game. You can use the public API of _CanvasModalWidget() to run your demos on it. The JavaScript (JS) variable that represents the modal is called "canvasModalWidget" in WebGLMainController.js. There is also a second service called "webGLDrawUtilities" in the same file. This service is used to generate the data (vertices, surface normals, vertex indexes) used by WebGL to render some 3D primitives used by the demos.
  • CSS/Image Demo Files Composition
    • styles/app/app.css: Contains the basic styling used for the authored content (overriding some of HTML5 Up's styling and introducing some new ones) and the modal styling used to present the WebGL demos.
    • styles/app/images: Contains the directory where all demo screenshots and 3rd party images are stored.

  • Process to Bootstrap your WebGL Application
    • Get a WebGL Context (i.e. var gl = canvas.getContext("webgl")).
    • Compile your Vertex and Fragment Shader using your context (i.e. var compiledShader = gl.compileShader(shaderString))
    • Get a reference to a program object from your context (i.e. var program = gl.createProgram()).
    • Use your context to attach both compiled shaders to your program object (i.e. gl.attachShader(compiledShader)).
    • Link your program with the context (i.e. gl.linkProgram(program)). Note that you can use multiple programs in your app.
    • Associate the program to use in your current context (i.e. gl.useProgram(program)).
