Introduction to WebGL

This page provides an introductory lesson to WebGL with a focus on providing you with the basic tools to building a fully functioning Pong game.

Skip to the Demos!

Learning Objectives

  • Understand the benefits/disadvantages of WebGL.
  • Learn about the files used by the WebGL demos. You will need this knowledge for the proceeding sections.
  • Learn about the history of WebGL.
  • Gain an understanding of the WebGL Pipeline.
  • Be able to explain the purpose and utilization of Vertex and Fragment shaders.
  • Be able to describe and apply the basic steps to initialize a WegGL application.
  • Be able to describe and apply the basic steps required to draw a basic primitive in WebGL.
  • Describe the cartesian coordinate space conventionally used by 2D Canvas.
  • Describe the cartesian coordinate space conventionally used by the WebGL Canvas.
  • Understand the difference between Model, View/Camera and Perspective Spaces.
  • Describe the two popular types of 3D projection used in WegGL.
  • Apply your understanding of Model, View/Camera and Perspective Spaces to position 3D objects in the WebGL coordinate space.
  • Understand the basics of lighting in your WebGL application.
  • Learn (using the supplied demos) how to write your own WebGL applications.

  • Process to Bootstrap your WebGL Application
    • Get a WebGL Context (i.e. var gl = canvas.getContext("webgl")).
    • Compile your Vertex and Fragment Shader using your context (i.e. var compiledShader = gl.compileShader(shaderString))
    • Get a reference to a program object from your context (i.e. var program = gl.createProgram()).
    • Use your context to attach both compiled shaders to your program object (i.e. gl.attachShader(compiledShader)).
    • Link your program with the context (i.e. gl.linkProgram(program)). Note that you can use multiple programs in your app.
    • Associate the program to use in your current context (i.e. gl.useProgram(program)).
